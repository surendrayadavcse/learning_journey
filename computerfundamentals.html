Computer Fundamentals

Introduction

Computers are electronic devices that process data and perform tasks based on instructions.

Software Development Life Cycle (SDLC)

The Software Development Life Cycle (SDLC) is a process used to design, develop, and test software.

Phases of SDLC:

  • Requirement Analysis - Understanding user needs.
  • Planning - Defining scope and resources.
  • Design - Creating architecture and UI.
  • Development - Coding the software.
  • Testing - Finding and fixing bugs.
  • Deployment - Releasing the software.
  • Maintenance - Updates and improvements.

Agile Methodology

Agile is an iterative approach that delivers software in small increments.

Key Agile Principles:

  • Customer satisfaction through continuous delivery.
  • Welcoming changing requirements.
  • Delivering working software frequently.
  • Collaboration between developers and business stakeholders.
  • Focus on simplicity and high-quality work.

Scrum Framework

Scrum is a popular Agile framework for managing projects.

Scrum Roles:

  • Product Owner - Defines the product vision and priorities.
  • Scrum Master - Facilitates the Scrum process.
  • Development Team - Delivers the product.

Scrum Artifacts:

  • Product Backlog - A list of features.
  • Sprint Backlog - Selected tasks for a sprint.
  • Increment - A working product version.

Scrum Events:

  • Sprint - A fixed time frame for work (1-4 weeks).
  • Daily Scrum - A short daily meeting.
  • Sprint Review - A meeting to review completed work.
  • Sprint Retrospective - A meeting to improve the process.
